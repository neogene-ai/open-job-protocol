# OJP v0.1 → v0.2 Migration Guide

## Overview

OJP v0.2 flattens the nested v0.1 structure into top-level fields with snake_case keys, aligns field names directly with [OTP v0.2](https://opentalentprotocol.org), and adds ESCO taxonomy support for skills. The schema enforces `additionalProperties: false` for reliable machine parsing.

## Structural changes

| Change | v0.1 | v0.2 |
|--------|------|------|
| **Nesting** | Grouped under `meta`, `role`, `organization`, `requirements`, `offering` | Flat top-level fields |
| **Casing** | Mixed (mostly camelCase IDs) | Consistent snake_case |
| **ID field** | `meta.job_id` (free-form string) | `ojp_id` (UUID format) |
| **Version** | `meta.version: "0.1"` | `schema_version: "0.2.0"` (semver) |
| **Status values** | UPPERCASE (`"ACTIVE"`) | lowercase (`"active"`) |
| **Skill matching** | `name` + `min_years` only | `name` + `min_level` (1–5) + `min_years` + `esco_id` |
| **Languages** | `code` + `level` | `language` (ISO 639-1) + `proficiency` (CEFR) |
| **Schema enforcement** | `additionalProperties` not set | `additionalProperties: false` everywhere |

## Field mapping

### Required fields

| v0.1 path | v0.2 field | Notes |
|-----------|-----------|-------|
| `meta.version` | `schema_version` | `"0.1"` → `"0.2.0"` |
| `meta.job_id` | `ojp_id` | Must be UUID format in v0.2 |
| `meta.status` | `status` | Lowercase: `"ACTIVE"` → `"active"` |
| `meta.created_at` | `created_at` | No change in format |
| `meta.updated_at` | `updated_at` | No change in format |
| `role.title` | `title` | Promoted to top level |
| `role.description` | `description` | Promoted to top level |
| `role.employment_type` | `employment_type` | Promoted to top level |
| `organization` | `organization` | Same structure, minus `logo_url` and `founded` |
| `offering.location` | `location` | `primary_location` → `city` + `country`; `alternate_locations` removed |

### Skill requirement changes

```
// v0.1
{ "name": "Go", "min_years": 3 }

// v0.2
{ "name": "Go", "min_level": 4, "min_years": 3, "esco_id": "S5.6.1" }
```

- `min_level` (1–5) maps to OTP `skills[].level` for numeric scoring
- `esco_id` enables taxonomy-based matching

### Language requirement changes

```
// v0.1
{ "code": "en", "level": "professional" }

// v0.2
{ "language": "en", "proficiency": "C1" }
```

- Field renamed from `code` → `language` (ISO 639-1)
- Free-text levels replaced with CEFR scale: A1, A2, B1, B2, C1, C2, native
- Matches OTP `languages[].proficiency` exactly

## Quick conversion checklist

1. Change `meta.version: "0.1"` → `schema_version: "0.2.0"`
2. Convert `meta.job_id` to UUID format → `ojp_id`
3. Lowercase all `status` values
4. Promote `role.*`, `meta.*` fields to top level
5. Flatten `offering.compensation.salary` → `salary_band`
6. Flatten `requirements` → `must_have` / `nice_to_have`
7. Update skill objects: add `min_level`, optionally `esco_id`
8. Update language objects: `code` → `language`, `level` → `proficiency` (CEFR)
9. Flatten `offering.location` → `location`, convert `primary_location` to `country` + `city`
10. Remove dropped fields: `extensions`, `visibility`, `locale`, `logo_url`, `founded`
